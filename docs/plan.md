# Nai2Android 实现计划

## 阶段一：基础工程与接口合同

- 建立可导入 Android Studio 的 Kotlin/Compose 工程。
- 固化 Nai2API 的请求参数、任务状态和图片地址模型。
- 建立 SQLite 数据表、加密密钥存储和 MediaStore 保存器。
- 验证：纯 Kotlin 单元测试先覆盖 tag 规范化、任务参数和图库排序。

## 阶段二：生成切片

- 实现设置页中的服务地址、访问密钥和余额查询。
- 实现生成表单、预设套用、任务轮询和图片下载。
- 生成成功后同时写入系统图库和本地索引。
- 验证：错误状态可见，图片下载失败不会产生“已完成”记录。

## 阶段三：图库切片

- 实现本地图片网格、详情页、时间排序、收藏和 tag 筛选。
- 图片详情支持修改归档 tag，并即时反映到筛选结果。
- 验证：重启应用后收藏、tag 和图片记录仍然存在。

## 阶段四：预设切片

- 实现预设列表、新增、编辑、删除。
- tag、artist、反向提示词均支持长文本和追加编辑。
- 验证：预设可套用并在生成前继续修改。

## 阶段五：交付检查

- 运行单元测试、静态检查和可用时的 debug 构建。
- 检查 APK 不含密钥，整理使用说明和已知限制。

## 主要风险

1. 站点接口属于第三方服务，返回字段或鉴权方式变化时需要同步适配。
2. 任务完成图片可能要求额外请求头或 token，下载器将同时支持站点返回的完整 URL 和相对 URL，并附带当前访问密钥。
3. Android SDK 未安装，APK 构建会受环境影响；源码工程与 JVM 单元测试可以先独立交付。
